## Plan: FillBlank Structured Math Keypad

Status: approved for implementation


在 fillblank 題型加入按鈕觸發的數學小鍵盤，但不是只插入單一符號，而是在彈出面板內提供輕量的結構化數學輸入器。建議做法是：主答案欄仍保留現有單一文字輸入框；按下數學鍵盤按鈕後，打開一個小型數學面板，面板內可建立真正的上下分數格、完整平方根外觀，以及常用乘除與未知數按鍵。使用者完成一個數學結構後，再把它序列化成一般字串，例如 `(a)/(b)`、`sqrt(x)`、`x×y`，插回主答案欄。這樣能滿足視覺輸入需求，同時避免把整個 fillblank 重寫成大型數學編輯器。

**Steps**
1. Phase 1 - 互動架構：在 `renderFillBlankInteraction_` 旁建立 fillblank 專用的鍵盤開關與彈出面板渲染流程，預設隱藏，按一下才展開。主輸入欄維持現有單一 `input`，數學面板則作為輔助編輯器。這一步先定義面板的開啟、收合、再次點擊切換、點擊外部關閉、切題重置與提交後關閉行為。
2. Phase 1 - 結構化分數編輯器：在彈出面板內加入「分數卡片」元件，畫面上要有中間橫線，上方一個分子輸入位，下方一個分母輸入位，符合使用者指定外觀。此元件不直接提交，而是在使用者確認後序列化成一般字串插回主答案欄。建議第一版序列化成 `(分子)/(分母)`，兼顧可讀性與現有答案比對流程。此步依賴步驟 1。
3. Phase 1 - 完整根號元件：在彈出面板內加入完整平方根外觀，而不是單一 `√` 字元。可做成「根號外框 + 內容輸入位」的小型元件，確認後序列化成 `sqrt(內容)` 或等價標準字串，再插回主答案欄。此步依賴步驟 1。
4. Phase 1 - 常用按鍵集：在彈出面板加入乘、除、百分比、π、次方、括號，以及 `x`、`y` 等未知數按鍵。這些按鍵可直接插入目前主輸入欄或面板內正在編輯的結構欄位，視當前焦點決定落點。此步依賴步驟 1。
5. Phase 1 - 焦點與插入規則：新增一套輕量的焦點管理，區分「主答案 input 正在編輯」與「彈出面板中的分子、分母、根號內容正在編輯」。當使用者在結構元件內輸入時，按鍵應作用於該欄位；按下確認後，再把序列化結果插入主答案欄的游標位置。此步依賴步驟 2 到 4。
6. Phase 2 - 樣式與版面：在 `數學小遊戲.html` 現有樣式區新增 math keypad 與 math builder 的樣式，至少包含開關按鈕、彈出面板、分數卡片、完整根號卡片、按鍵 grid。面板需靠近輸入框，手機版不能遮住主要提交按鈕，並維持足夠大的觸控面積。此步可與步驟 5 平行，但完成前需核對 DOM 結構一致。
7. Phase 2 - 主輸入整合：把現有 fillblank 的 `input` 監聽與新面板整合，保持既有 Enter 提交、空值禁用提交按鈕、以及提交後停用控制項的行為不變。面板插入內容後必須手動觸發與使用者輸入一致的更新流程，避免提交按鈕狀態失真。此步依賴步驟 5。
8. Phase 2 - 答案正規化：擴充 `normalizeAnswerText_`，統一處理由新面板產生的常見格式，例如 `π` 與 `pi`、`×` 與 `*` 或 `x` 的可接受規則、`÷` 與 `/` 的統一、`sqrt(...)` 與可能的 Unicode 根號寫法、百分比的全形半形差異。這一步的目標不是完整數學解析，而是確保新輸入器輸出的格式與題庫常見答案能穩定比對。此步依賴序列化格式確定。
9. Phase 3 - 邊界條件：確認切到下一題、提交答案、超時、自動停用、重新載入題目時，彈出面板、分數格內容、根號內容、主輸入游標與開啟狀態都會正確重置，不會殘留上一題的編輯狀態。此步依賴步驟 1 到 8。
10. Phase 3 - 文案與引導：把 fillblank 的提示文案微調為提示可點擊按鈕開啟數學小鍵盤，必要時在面板中加一個簡短提示，說明分數與平方根需先在面板內完成，再插入答案欄。這一步可最後完成。

**Verification**
1. 手動驗證 fillblank 題型：確認數學鍵盤預設隱藏，按開關按鈕才彈出，再次點擊或點擊外部可收合；桌面與手機寬度都能正常操作。
2. 手動驗證分數元件：確認可在彈出面板中看到真正的上下分數格與中間橫線，分子和分母各自可輸入；按確認後，會以預定字串格式插回主答案欄正確位置。
3. 手動驗證根號元件：確認畫面呈現完整根號外觀，根號內有輸入位；按確認後，會以預定字串格式插回主答案欄。
4. 手動驗證按鍵集：依序測試 `×`、`÷`、`%`、`π`、`x`、`y`、次方、括號、刪除、清空，確認在主輸入欄與結構元件內都會落到正確位置。
5. 手動驗證答案比對：用題庫或臨時樣例測試 `pi`/`π`、`(1)/(2)` 與 `1/2`、`sqrt(4)`、`50%`、含 `x` 或 `y` 的答案，確認新正規化不會讓原本簡單數字答案退化。
6. 手動驗證流程：提交正確與錯誤答案、超時、切到下一題後，檢查控制項是否依既有流程停用，且彈出面板與面板內暫存內容同步清空。
7. 若環境已有可直接開啟的前端預覽或 Apps Script web app，使用該最小路徑實測一輪 fillblank 題目；若沒有自動化測試框架，本次以針對性手動驗證為主。

**Decisions**
- 已確認：小鍵盤改為按一下才彈出，而不是固定顯示在 fillblank 輸入框下方。
- 已確認：分數需要視覺上的上下兩格與中間橫線，不接受只插入單一 `/` 字元作為主要交互。
- 已確認：平方根需要顯示完整根號外觀，而不是只插入單一 `√` 字元。
- 已確認：需加入乘、除、`x`、`y` 等未知數相關按鍵。
- 已確認：內部提交與答案比對採用一般字串格式，例如 `(a)/(b)`、`a/b`、`sqrt(x)`，不以 LaTeX 作為主要提交格式。
- 建議採用：主答案欄維持現有單一文字輸入框；分數與根號在彈出面板內以小型結構化編輯器完成後，再序列化插回主答案欄。這是目前程式架構下成本最低、風險最可控的做法。
- 包含範圍：fillblank 題型前端輸入體驗、彈出層狀態管理、分數與根號結構元件、常用數學按鍵、基本答案正規化、響應式樣式。
- 不包含範圍：後端題庫結構變更、完整數學語法解析器、整個 fillblank 升級成通用數學編輯器、其他題型的鍵盤擴充。

**Further Considerations**
1. 乘號 `×` 與未知數 `x` 在視覺上容易混淆，建議按鍵樣式與正規化規則刻意區分，例如乘號按鍵固定輸出 `×`，未知數按鍵固定輸出小寫 `x`。
2. 若題庫現有答案大量使用 `1/2` 而不是 `(1)/(2)`，正規化層應把兩者視為等價，否則會出現新輸入器可輸入但舊答案比對失敗的問題。
3. 若之後想讓學生在主答案欄直接看到漂亮分數與根號，而不只是序列化字串，就會進一步演變成真正的數學編輯器；那會是下一個獨立階段，不建議和本次需求混在同一次改動。
